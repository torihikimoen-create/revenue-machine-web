# AI未経験層・非エンジニア層向けアウトリーチ戦略

AIという言葉に抵抗がある、または使い方が想像できない層に対し、AETHER COREがどのようにアプローチするかを定義します。

## 1. 核心的コンセプト：AIを「消す」
「AIを導入しましょう」ではなく「業務の苦痛を消しましょう」というメッセージに統一します。

| 一般的な表現（AI層向け） | 優しい表現（未経験層向け） |
| :--- | :--- |
| AIによる自動化 | 業務の丸投げ、自動片付け |
| 生成AI、LLM | デジタル分身、24時間稼働の事務スタッフ |
| プロンプトエンジニアリング | 業務指示の最適化（弊社が代行） |
| 業務効率化 | 残業ゼロ、家族との時間の確保 |

## 2. 推奨されるメールのタイトル例
ターゲットの「最も嫌いな作業」をタイトルに入れます。

- **産廃・建設**: `【ご提案】現場から戻った後の「日報・マニフェスト作成」を自動化しませんか？`
- **医療・介護**: `【相談】夜勤後の「記録入力」による残業を月20時間減らす具体策`
- **事務・経理**: `メールの山とデータの転記を「自動で終わらせる」新しい仕組みのご案内`
- **共通**: `【人手不足対策】月額3万円台で、24時間働く「優秀な分身」を雇う方法`

## 3. 「使い方の不安」を解消する3つの約束
メール本文やLPで以下の3点を強調します。

1.  **初期設定不要**: 弊社がすべて構築します。お客様は「いつもの作業」を教えるだけです。
2.  **操作不要**: AIの操作は必要ありません。結果が勝手にメールやチャットに届きます。
3.  **いつでも解約**: 10日間の無料期間で「本当に楽になるか」をノーリスクで試せます。

## 4. システムへの実装指針
- `target` オブジェクトに `awareness`: `low`（未経験層）フラグを新設。
- `low` フラグが立っている場合、`generator.py` 等で「テクノジー用語禁止プロンプト」を適用し、上記タイトル・本文を生成させる。
