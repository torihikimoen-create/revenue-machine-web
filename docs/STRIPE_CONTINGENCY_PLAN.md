# Stripe 審査不通過・追加確認への対応ガイド & テンプレート

Stripe から審査結果（不通過、または追加情報の提供依頼）が届いた際、迅速かつ的確に再申請を行うための準備資料です。

## 1. 想定される修正・対応パターン

審査に落ちる場合の主な理由は「ウェブサイト上の情報不足」がほとんどです。

| 落ちた理由（想定） | 必要なアクション |
| :--- | :--- |
| **特商法表記の不足** | 住所・電話番号・メール・販売価格・支払い時期の明記（修正済みだが再点検） |
| **返金・キャンセル規定不足** | サービスの性質上、返金不可である旨や解約条件の明記 |
| **ビジネスモデルの不明瞭** | 「何をして対価を得るか」のさらなる具体化（AIによるB2B事務効率化など） |
| **本人確認・実在確認** | 公的身分証の再アップロードや、公共料金の領収書の提出 |

## 2. Stripe への再審査依頼・回答テンプレート

### パターンA：ウェブサイトの修正を完了し、再審査を依頼する場合
（Stripe のダッシュボードまたは返信メールで使用）

> **件名**: 【再審査依頼】ウェブサイトの修正完了および追加情報について (アカウントID: [Your_Account_ID])
>
> **本文**:
> Stripe 担当者様
>
> いつもお世話になっております。
> この度は、アカウントの審査およびフィードバックをいただき、誠にありがとうございます。
>
> 指摘事項に基づき、弊社ウェブサイト（[Your_Website_URL]）にて以下の修正・更新を行いました。
>
> 1. **特定商取引法に基づく表記の更新**: 
>    住所、連絡先、および支払い時期について、より詳細かつ正確な情報を追記いたしました。
> 2. **キャンセル・返金ポリシーの明文化**: 
>    デジタルサービスの特性に応じた返金規定および解約手順について、規約ページに明記いたしました。
> 3. **ビジネス概要の補足**:
>    弊社の提供する「AETHER CORE」は、中小企業向けのB2B事務作業自動化およびリード獲得支援サービスです。具体的な提供価値（レポート作成、事務プロセス自動化）について、トップページでより詳細に説明しております。
>
> 以上の修正により、Stripeの利用規約および日本の各法律に準拠した状態となっております。
> お忙しいところ恐縮ですが、再度アカウントの有効化についてご検討いただけますと幸いです。
>
> 追加の資料（事業実態を示す資料など）が必要な場合は、即座に提出させていただきます。
>
> 何卒よろしくお願い申し上げます。
>
> [あなたの名前/法人名]

> 弊社は実態のあるB2Bサービスを運営しており、不適切な商取引は一切ございません。

## 3. チェックリスト（送信前の最終確認）

- [ ] サイトのフッターから「特商法に基づく表記」に1クリックで飛べるか？
- [ ] サイト上の住所と、Stripe に登録した住所が1字1句違わないか？
- [ ] サイト上に「テスト中」「工事中」などの未完成を想起させる記述がないか？
- [ ] 販売価格が「見積もり」だけでなく、最低価格や料金体系が示されているか？
